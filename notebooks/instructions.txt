TOKEN BUDGETS (SAFETY DURING DEVELOPMENT)

The backend enforces optional OpenAI token caps stored in Redis: a rolling per-UTC-day limit (total_tokens reported by the API, with conservative pre-flight estimates before each call) and a per-scope limit (one pending enrichment job, one claim-analysis action, one search request embedding, or one approve-pending embedding batch). Defaults are conservative for local dev; set OPENAI_ENFORCE_TOKEN_BUDGETS=false to disable, or raise OPENAI_MAX_TOKENS_PER_DAY / OPENAI_MAX_TOKENS_PER_CLAIM_SCOPE (0 disables that cap). AI PIPELINE RULES

======================================================================
RULE 1 — EMBEDDINGS FIRST

Use embeddings BEFORE expensive reasoning.

Workflow:

1. User submits claim
2. Generate embedding ONCE
3. Store permanently in PostgreSQL pgvector
4. Use vector search for:
- duplicate detection
- semantic retrieval
- related claims
- contradiction candidates

Store:

- embedding vector
- embedding model
- embedding version
- embedding timestamp

NEVER regenerate embeddings unnecessarily.

======================================================================
RULE 2 — CACHE EVERYTHING

Use Redis caching.

Cache:

- AI summaries
- duplicate detection results
- semantic retrieval outputs
- confidence analyses
- evidence retrieval outputs

DO NOT rerun AI if:

- evidence unchanged
- moderation unchanged
- freshness not expired

======================================================================
RULE 3 — ASYNC AI ONLY

All expensive AI tasks MUST run asynchronously using Celery workers.

Submission flow:

- claim stored instantly
- response returned immediately
- enrichment handled in background

NEVER block user requests waiting for AI reasoning chains.

======================================================================
RULE 4 — RETRIEVAL FIRST

Workflow:

1. Search existing claims via pgvector
2. Search existing evidence
3. Retrieve relevant context
4. ONLY THEN call GPT

GPT MUST operate on retrieved context.

GPT MUST NEVER hallucinate unsupported facts.

======================================================================
RULE 5 — LIMITED WEB RETRIEVAL

DO NOT implement autonomous crawling agents.

Web retrieval only occurs:

- during enrichment
- during scheduled refreshes
- when moderators request re-analysis

MVP retrieval sources:

- trusted RSS feeds

- public APIs
- manually submitted URLs

Preserve:

- original URL
- retrieval timestamp
- retrieval source
- provenance

RULE 6 — CHEAP MODELS FOR CHEAP TASKS
